Notes

Alignment

If we need to center a block element, we can use align="center/left/right"

If we need to center an inline element (or inline-block) we can use text-align="center"

valign is used to with TD's to vertically align its content (which can be another table). When using valign to align content, also add the css equiv, vertical-align

Outer TD's should have padding removed. Add it back in as necessary.

Outer tables should have border spacing 0 and width 100%;

Inner TD's can be used to add backgrounds, padding, alignment of content, and width, either pixel or percentage, and any other style you need (font, etc)

Banners

Inside Banner's, the tables will need cellpadding and cellspacing of 0.

A banner with text will go on a td as a background="https://yadayada" with explicit width and height attributes that match the dimensions of the image.

The td should also have a css att of background-position:"center top"

Outlook can't do background images, so we need a Conditional Statement to handle this for us. It's best just to copy the code.

Also, directly after your banner td, you will have 3 nested tables.

All 3 with a class of darkmode-transparent that will make our backgrounds transparent on darkmode (otherwise, they will shine right through the image on outlook). That should just say background-color: tranparent !important

Table 1: full frame. Td will be the full width of the email (width="600") and will vertically and horizontally center our next table (which will wrap the content table) with align="center" and valign="middle" on our td.

Table 2: centering table. Has no width and so will expand to the content size. The td will center align the content table. It's purpose is just to be in the middle, and house the content table. This also can have multiple rows if we want to separate our content into rows, with different styling or purposes, like a button for example.

Table 3: The final table wraps the content. The td can be used for alignment, and padding.

To have space in between columns, we can just add a column with no content, give it a height, and a disappearing class on small screens

In order to get our columns to stack appropriately, we need to add a font-size of 0 on the column wrapper td. This will eliminate the inherent font-soze that td's have, in order to make our calculation accurate for stacking.
